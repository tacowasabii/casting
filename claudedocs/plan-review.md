## 🔴 A. 간단 MVP라도 빼기 어려운 것 (전부 합쳐 1~2일 분량)

실지원자(불특정 다수)의 개인정보가 들어오는 순간 걸리는 것들. 지인 데모 단계(8주차)까지는 필요 없고,
**파일럿에서 실공고에 폼 링크를 붙이기 전**에만 하면 됨.

| # | 항목 | 최소 구현 | 비용 |
|---|------|-----------|------|
| A1 | 지원 폼 동의 2단 분리 | 필수: "본 배역 검토 목적, 배역 종료 시 파기" / 선택: "인재풀(라이브러리) 등록, 보유 3년, 미동의해도 지원 가능" 체크박스 | 체크박스 2개 + 동의 컬럼 1개 |
| A2 | 감독 공유 고지 | 동의문에 한 문장: "제출한 프로필·영상은 해당 작품 제작진에게 캐스팅 검토 목적으로 공유됩니다" (별도 동의 체크박스는 과잉 — 불필요) | 문구 1줄 |
| A3 | 아역(만 14세 미만) 분기 | 생년 기준 판별 → v1은 "만 14세 미만은 보호자가 지원해 주세요" 안내 후 차단으로 시작해도 됨 | if문 + 안내 화면 |
| A4 | 리뷰 링크 데이터 최소화 | 감독 뷰에 연락처(전화·이메일·소속사 연락처) 절대 미포함 + 만료일을 옵션이 아닌 기본값(예: 마감+7일)으로 | 필드 화이트리스트 + 기본값 |
| A5 | 처리방침 정적 페이지 1장 | 수집 항목·목적·보유기간·위탁 업체(Supabase/Mux 등)·삭제 요청 이메일. 폼 하단 링크 | 반나절 |

**미동의(A1 선택 항목) 지원자의 배역 종료 시 파기 배치**는 파일럿 중엔 수동 삭제로 대체 가능 — 자동화는 유료화 시점으로.

## 🟡 B. 코드가 아니라 "방향 결정" — 지금 정하면 재작업을 막는 것

구현할 필요는 없고, 어느 쪽으로 갈지만 정해두면 되는 것들.

- **B1. 영상 제출은 링크(유튜브·드라이브) 우선, 파일 업로드는 Phase 2에서 Mux Direct Upload와 함께.**
  Phase 1에 파일 업로드를 넣으면: ① 모바일 수백 MB 업로드 실패 = 지원 포기, ② Vercel 바디 한도(4.5MB)로 direct upload 강제, ③ 4주차에 Supabase Storage → Mux 저장 경로 재작업. 링크 우선이면 Phase 1 폼이 하루짜리로 단순해짐.
- **B2. 이메일 인박스(Postmark 파싱)는 12주 밖으로. 파일럿은 폼 단독 + 수동 등록 UI.**
  파싱은 엣지케이스 최다 구간(Postmark 10MB 한도로 영상 첨부 수신 불가, 네이버 대용량첨부 만료 링크, 한글 파일명 인코딩, 소속사 일괄 메일). 이메일로 새는 지원자는 디렉터가 손으로 넣는 수동 등록이 안전망 겸 대체재.
- **B3. Actor 자동 병합 안 함.**
  "발신 주소 기준 자동 병합"은 소속사 매니저가 한 메일로 배우 N명 보내는 흔한 케이스에서 서로 다른 배우를 하나로 합쳐버림. 간단 MVP에선 **그냥 병합 로직 자체를 만들지 않는 게** 정답 — 전화번호 일치 시 "동일 인물?" 안내만 띄우고 나머지는 수동. 병합 기능은 라이브러리가 실제로 쌓인 뒤에.
- **B4. 비로그인 경로(폼 제출·리뷰 링크) 원칙 한 줄: anon 키로 클라이언트에서 직접 테이블 접근 금지, 전부 Route Handler + service role 경유.**
  Supabase 기본 패턴을 그대로 쓰면 익명 쓰기 경로에 구멍이 나기 쉬움. 원칙만 지키면 추가 비용 거의 없음.
- **B5. 리뷰 링크는 v1에서 "링크당 리뷰어 1인" 가정으로 명시하고 세션 식별은 만들지 않음.**
  감독+조연출이 같이 열면 응답이 섞이는 문제가 있으나, 파일럿 감독 1인 기준으론 발생 안 함. 가정만 문서에 적어두면 됨.

## 📋 D. 구현과 무관 — 검증 설계 메모 (코드 0줄, 비용 0)

MVP 구현 범위와 별개로, 만들기 **전에** 하면 12주의 방향 자체를 바꿀 수 있는 것들.

1. **0주차 인터뷰**: 지인 디렉터의 최근 배역 실측 — 실제 지원 건수, 엑셀 전사 시간, 감독 컨펌 왕복 횟수. 3절 페인 등급("추정")을 데이터로 교체. 이게 틀리면 Phase 2~3 투자 배분이 바뀜.
2. **표본 확인**: 지인의 업무에서 공고 기반 조단역 비중이 실제로 큰지 + 10~14주차에 진행되는 공고 기반 배역이 있는지. 없으면 파일럿이 핵심 가치를 테스트하지 못함.
3. **성공 기준 ③ 교체**: 지인의 구두 금액 응답은 데이터가 아님. 파일럿 성공 시 명목가(월 2~3만 원) 계좌이체 요구 → 결제 여부를 기준으로.
4. **재사용 기준 통일**: "다음 프로젝트"(2절) vs "다음 배역"(8절) → **동일 작품 내 다음 배역**으로 통일해야 14주 안에 관측 가능.
5. **파일럿 유효 기준**: "배우가 폼으로 지원 안 함" 리스크 — 폼 경유 지원 최소 건수(예: 30건)를 정해두고, 미달 시 도구 문제가 아니라 채널 문제로 판정.
